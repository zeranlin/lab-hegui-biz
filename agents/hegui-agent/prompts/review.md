# 招标文件审查任务提示词

请对指定的政府采购招标文件执行合规性审查生产任务。

## 输入

- 文件路径：`{{FILE_PATH}}`
- 文件分类：`{{CATEGORY}}`
- 可读范围：`{{WORKSPACE}}`
- 本次审查产物输出目录：`{{OUTPUT_ROOT}}`

## 执行动作

1. 校验文件位于本项目或只读知识目录内，且不得读取同一项目既有审查记录作为答案来源。
2. 读取或生成临时抽取文本；如需持久化，只能写入审查产物输出目录。
3. 按 `wiki/00-入口/外部执行主体招标文件审查指引.md` 的顺序读取必调知识，建立审查口径，不得修改知识目录。
4. 建立文件画像：项目编号、名称、采购人、采购方式、评标方法、预算、品目、所属行业、场景标签、政策适用、章节完整性。
5. 命中物业管理后，必须执行 `wiki/70-审查协议/物业管理审查动作协议.md` 中 `PROP-A01` 至 `PROP-A23` 的全部动作。
6. 每个动作状态只能是 `命中`、`未命中` 或 `待确认`；未命中也要在运行记录中说明已读章节和不成立原因。
7. 按章节审查招标公告、资格/符合性、采购计划和预算、政府采购政策、用户需求书、评分办法、证明材料、投标文件格式、合同条款。
8. 按 `wiki/70-审查协议/风险原子化规则.md` 拆分风险，不得把应拆分风险合并成一条。
9. 对照风险审查点输出风险清单，每条风险必须包含关联动作ID和关联审查点。
10. 执行 `wiki/70-审查协议/质量门规则.md`，特别反查人员评分、供应商能力、用户需求书、合同条款、方案和承诺评分。
11. 风险数量明显偏低时，必须执行二次反查并写入运行记录。
12. 在 `{{OUTPUT_ROOT}}/` 下生成 `xxx-审查报告.md`，不得写入本次输出目录之外。
13. 在 `{{OUTPUT_ROOT}}/` 下生成 `xxx-AI调度运行记录.md`，不得写入本次输出目录之外。
14. 如生成抽取文本或临时辅助产物，也必须写入 `{{OUTPUT_ROOT}}/` 下，不得覆盖历史产物。
15. 不对只读知识目录执行 `./lint`、`./ingest`、`./query` 或任何维护命令。

## 审查记录时间要求

审查报告顶部必须同时记录：

```text
审查日期:: {{REVIEW_DATE}}
审查时间:: HH:MM:SS CST
```

`审查时间` 使用执行当下的具体时间点，精确到秒。

## 风险输出格式

每个风险点使用：

```text
### 编号. 风险标题

- 结论类型：
- 风险等级：
- 原文位置：
- 原文内容：
- 问题说明：
- 关联动作ID：
- 关联审查点：
- 审查依据：
- 修改建议：
- 是否需要人工复核：
```

## 禁止事项

- 不得使用外部 V2、标准答案或人工标注。
- 不得把问题说明写进 `原文内容`。
- 不得只凭标题或摘要生成风险，必须有原文证据。
- 不得生成第二套“校准版”审查记录。
- 不得读取同一项目既有审查记录或运行记录作为答案来源。
- 可以读取外部执行主体指引、审查协议、动作协议、风险原子化规则、质量门规则和同品类样板规则；不得复制同一项目历史风险清单。
- 不得删除既有项目记录。
- 不得写入或更新只读知识目录。
- 不得生成知识更新申请、人工复盘记录或来源笔记。
- 不得对只读知识目录运行维护命令。

## 运行记录要求

运行记录必须包含：

- 调用知识清单；
- 文件画像；
- `PROP-A01` 至 `PROP-A23` 动作清单和执行状态；
- 原文读取记录；
- 风险原子化检查；
- 质量门结果；
- 未命中动作说明；
- 二次反查说明；
- 是否使用外部标注；
- 是否修改 LLM Wiki；
- 是否对 LLM Wiki 运行维护命令。
